# Product Studio Portfolio Operating Model

## When to use this

Use when running multiple venture/client bets with shared platform capabilities.

## Model

A product studio is a **portfolio operator**: it allocates capital/talent across ventures, not just project teams.

## Operating model

1. **Opportunity pipeline**
   - Source opportunities from market pull, founder insight, and distribution advantage.
2. **Venture thesis scorecard**
   - Score on pain intensity, willingness to pay, time-to-validation, distribution access.
3. **Stage-gate progression**
   - Gate 1: evidence of problem urgency
   - Gate 2: solution behavior + willingness-to-pay signal
   - Gate 3: repeatable acquisition/retention loop
4. **Shared platform services**
   - Centralize legal, finance, growth ops, design system, data instrumentation.
5. **Capital allocation review**
   - Monthly reallocation toward highest evidence-adjusted upside.

## Portfolio checklist

- Every venture has explicit stage and next gate criteria
- Kill criteria defined before major spend
- Shared services cost allocated transparently
- Founding teams measured on learning velocity + traction
- Portfolio review includes downside scenario planning

## KPIs

- Time from idea to first validation milestone
- Cost per validated venture
- % ventures passing each stage gate
- Portfolio concentration risk (revenue/exposure by venture)
- Studio-level cash burn and runway

## Failure modes

- Treating all ventures equally despite uneven evidence
- No kill discipline (zombie ventures)
- Shared team context-switching overload
- Poor governance around equity/scope/incentives
